What Changed

Partitioned Hyperconnections

Why It Works

A generalization of skip connections where the residual stream is partitioned into segments with learned mixing coefficients. More expressive than simple skip connections.

Key Concepts to Understand

  • Partitioned hyperconnections
  • Residual stream partitioning
  • Learned mixing coefficients

▶ PR #230: [New Record] Parallel Residuals (-45 steps, -0.85s) +64390 / -36 in 17 files — by @msisovic

Adding parallel residuals (for lack of a better name) - attention and MLP read from separate residual streams, but write their output to both streams using learned lambdas. ``python if block.attn is not None: layer_out = block.attn(norm(lane0), attn_args, qkvo_w) layer_out = layer_out + skip_bias[i] lane0 = resid_lambda[si] * lane0 + layer_out * w_post0[si] lane1 = resid_lambda[si] * lane1 + layer_out * w_post1[si] ` `python if block.mlp is not None: layer_out = block.mlp(norm(lane1), c_fc, c_proj) lane0 = resid_lambda[si+1] * lane0 + layer_out * w_post0[si+1] lane1 = resid_lambda[si+1] * lane1 + layer_out * w_post1[si+1] ` This is only applied from block 7 and onwards. This is a tradeoff I made because the the lanes add significant memory bandwidth overhead since we double the amount of lanes we read/write to - while running experiments I noticed the delta between the write strengths between lane0 and lane was significant only in the later layers.

Inspiration

I converged to this architecture after trying to implement static hyperconnections from the bytedance paper. However, not only was the overhead huge, but they didn't help convergence at all using the initialization method proposed by the paper (basically starting off equivalent to pre-norm, all lanes symmetrical). While experimenting, a "round-robin" initialization where I force each layer to pick a favorite lane from the start showed promises, but not that much and the computation/memory overhead was still huge. I started experimenting which new parameters I could freeze and it turned out that freezing everything except the H_post matrix (the one determining how much of the layer output is mixed back into every lane) not only sped steps up, but boosted convergence. Another thing that turned out different from the paper was that 2 lanes was the best in my experiments. That lead me to simplify the architecture to attn and mlp having separate lanes, which was just an intuition leap since we had 2 lanes anyway.
